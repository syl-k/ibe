# M4 — 状態復元

> 目的: アプリ再起動時に、レイアウト構成・タブ・各ブラウザペインの URL を復元する。
> ターミナルのプロセスは復元しない（ペインの枠は復元するが shell は再起動）。
> 最終更新: 2026-07-01

---

## 1. 実装したもの / 検証結果

| 機能 | 状態 |
|------|------|
| レイアウトツリーの復元（分割・比率・ペイン種別） | ✅ 分割比率 0.697 まで復元 |
| ブラウザペインの URL 復元 | ✅ github.com / example.com |
| 複数タブの復元（タブごとに別レイアウト） | ✅ WS1=4分割 / WS2=単一 |
| アクティブタブの復元 | ✅ WS1 がアクティブで復帰 |
| ターミナルは枠のみ復元・shell は再起動 | ✅ 復帰後は新しいプロンプト |
| 起動時のちらつきなし（デフォルト描画を挟まない） | ✅ 初回描画前に hydrate |

## 2. 設計

### 保存先 / 責務分離
- `userData/session.json` に `{ version, tabs, activeTabId }` を保存。
- **main は「素の JSON ストア」**。`session:load`（読み）と `session:save`（500ms デバウンス書き、
  `before-quit` で flush）だけを担い、中身のスキーマは関知しない。
- **スキーマ検証は renderer が担う**（レイアウトツリーは renderer が所有するため）。
- `persist.ts`（M3 で導入）の `loadJson/saveJson` を再利用。

### 保存タイミング
- `main.tsx` で store を購読し、`serializeSession(state)` のスナップショットが変化したときのみ
  `ibe.session.save` を呼ぶ。タイトルだけの更新（`viewState` のみ変化）はスナップショットに
  出ないので保存を誘発しない。URL・分割・タブ・比率の変化で保存される。

### 復元タイミング（ちらつき回避）
- `main.tsx` の `boot()` で **最初の描画前に** `session.load()` を待ち、`hydrate()` してから
  `ReactDOM.createRoot().render()`。デフォルト 4 分割を一瞬描いてから差し替える、という
  無駄なビュー生成・ちらつきを避ける。

### シリアライズ / ハイドレート
- 保存時: ターミナル葉から **pty セッション情報を除去**（`stripSessions`）。プロセスは復元
  しないので保存不要。ブラウザ葉は url を保持。分割ノードは比率・向き・子を保持。
- 復元時:
  1. 保存データを検証（`version`・`tabs`・`activeTabId`・各ノード形状）。不正なら破棄して
     デフォルト継続。
  2. **id カウンタを復元 id の最大値まで前進**（`reseedCounter`）。復元後に採番する新 id が
     既存 id と衝突しないようにする（カウンタは毎起動 0 始まりのため必須）。
  3. 各ターミナル葉に **新しい単一セッションを付与**（`freshSessions`）→ 復帰後は新しい shell。

## 3. 既知の制約 / 見送り

- **キーボードショートカットは web ペインにフォーカスがあると効かない**（ネイティブ
  `WebContentsView` がキー入力を奪う）。恒久対応はアプリメニュー（Accelerator）登録が必要で、
  別途対応とする。本マイルストーンでは chrome/DOM にフォーカスがあれば動作。
- ウィンドウ自体の位置・サイズの復元は未対応（レイアウト内部のみ）。
- 履歴の巻き戻し状態（各ペインの前進/後退スタック）は復元しない（URL のみ）。
- スクロール位置・フォーム入力は復元しない。

## 4. 動作確認の要点

1. 4 分割の一部を分割/種別変更、ブラウザで別 URL に遷移、分割比率をドラッグ、タブを追加、
   アクティブを WS1 に戻す。
2. 再起動 → タブ構成・各レイアウト・比率・URL・アクティブタブが復元。ターミナルは新しい shell。
